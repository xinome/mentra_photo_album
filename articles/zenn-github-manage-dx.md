# はじめに

Zennの記事をGitHub管理に移行しました。

きっかけは、

- 記事をローカルで管理したい
- コードと一緒に履歴を追いたい
- 記事更新を怖がらずにできる状態を作りたい

という理由からです。

本記事では、

- 既存記事のslugを維持したまま移行する方法
- Zenn × GitHub連携の設定手順
- 実際に運用して感じたメリット
- おすすめの管理方法

をまとめます。

---

# 前提条件

- すでにZennアカウントを持っている
- GitHubアカウントを持っている
- 既存記事がZenn上に公開済み
- slug（URL末尾）を維持したい

---

# 全体の流れ

1. GitHubリポジトリを準備
2. Zenn CLIを導入
3. 既存記事をエクスポート
4. リポジトリに取り込み
5. pushして同期確認

---

# 1. GitHubリポジトリの準備

今回は**既存のアプリケーションリポジトリと連携**しています。

```
[リポジトリ名]/
├── app/
├── components/
├── articles/   (記事)
├── books/　　　 (本)
```

Zenn用に `articles/` 、　`books/` ディレクトリを設けています。

※ 専用リポジトリに分ける方法もありますが、今回は「実装と記事を同一管理する」方針にしました。

---

# 2. Zenn CLIの導入

```bash
npx zenn init
```

これで以下が生成されます。

```
articles/
books/
```

※ コマンドはリポジトリ直下で実行してください。

⸻

# 3. 既存記事の移行（slug維持）

## 重要ポイント

** ファイル名 = slug **

既存記事のURLを維持したい場合は、
エクスポートされたファイル名を変更しないことが重要です。

## 手順
	1.	Zennのエクスポート機能でzipを取得
	2.	zipを展開
	3.	articles/ 配下のファイルをそのままリポジトリへコピー

例：

```
export/articles/f06597757d6750.md
↓
repo/articles/f06597757d6750.md
```



---

# 4. pushして同期確認

```bash
git add .
git commit -m "Import existing Zenn articles"
git push origin main
```

push後、Zenn側で同期状況を確認します。
- 新規記事として増えていない
- 既存記事が更新扱いになっている

これが確認できれば成功です。

---

# 運用して感じたメリット

## 1. 記事の更新が怖くなくなる

Gitで履歴が残るため、大きなリライトも安心してできます。

## 2. 実装と記事が同期できる

コード変更と同時に記事を更新できるため、
- 実装内容
- 設計意図
- 解説

を常に一致させることができます。

## 3. DXの可視化になる

記事をGitHubで管理していること自体が、設計思想や開発スタイルの表現になります。

---

# おすすめの管理方法

## ① 記事は実装と同一リポジトリで管理

メリット：
	•	実装とドキュメントの一体化
	•	PRベースのレビューが可能
	•	変更履歴が明確

## ② 記事ファイル名は変更しない

slug維持のため、
既存記事はファイル名を変更しないこと。

## ③ ブランチ運用をする

例：

```
feature/zenn-rls-extend
```

記事と実装を同時に編集し、PRでまとめて管理する。

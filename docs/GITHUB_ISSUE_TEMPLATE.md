# GitHub Issue テンプレート

## 📝 Issue作成時の形式

以下の形式に従ってIssueを作成してください。

```markdown
## 📝 概要

[実装・修正内容の簡潔な説明]

## ✅ 実装内容

### 1. [実装項目1]

* [具体的な実装内容]
* [技術的な詳細]

### 2. [実装項目2]

* [具体的な実装内容]
* [技術的な詳細]

## 📁 変更ファイル

* `[ファイルパス]` - [変更内容の説明]
* `[ファイルパス]` - [変更内容の説明]

## ✅ 検証項目

* [ ] [検証項目1]
* [ ] [検証項目2]
* [ ] [検証項目3]

## 📚 関連Issue

* [関連Issueへのリンク（あれば）]
```

## 📋 Issueタイプ別のテンプレート

### 🐛 バグ修正

```markdown
## 🐛 バグの概要

[バグの簡潔な説明]

## 🔍 再現手順

1. [手順1]
2. [手順2]
3. [手順3]

## 💡 期待される動作

[期待される動作の説明]

## 🛠️ 修正内容

* [修正内容1]
* [修正内容2]

## 📁 変更ファイル

* `[ファイルパス]` - [変更内容]

## ✅ 検証項目

* [ ] バグが再現しないこと
* [ ] 既存機能に影響がないこと
```

### ✨ 新機能実装

```markdown
## ✨ 機能の概要

[新機能の簡潔な説明]

## 📋 要件

* [要件1]
* [要件2]

## ✅ 実装内容

### 1. [実装項目1]

* [具体的な実装内容]
* [技術的な詳細]

## 📁 変更ファイル

* `[ファイルパス]` - [変更内容の説明]

## ✅ 検証項目

* [ ] [機能1が動作すること]
* [ ] [機能2が動作すること]
* [ ] レスポンシブ対応が正しいこと
```

### 🔧 リファクタリング

```markdown
## 🔧 リファクタリングの概要

[リファクタリング内容の簡潔な説明]

## 🎯 目的

* [目的1]
* [目的2]

## ✅ 変更内容

* [変更内容1]
* [変更内容2]

## 📁 変更ファイル

* `[ファイルパス]` - [変更内容]

## ✅ 検証項目

* [ ] 既存機能が正常に動作すること
* [ ] パフォーマンスが改善されていること（該当する場合）
```

### 📝 ドキュメント整備

```markdown
## 📝 ドキュメント整備の概要

[ドキュメント整備内容の簡潔な説明]

## ✅ 追加・更新内容

* [ドキュメント1] - [内容]
* [ドキュメント2] - [内容]

## 📁 変更ファイル

* `[ファイルパス]` - [変更内容]

## ✅ 検証項目

* [ ] ドキュメントが正しく表示されること
* [ ] 内容が正確であること
```

## 📌 ラベル付けの推奨

* `bug` - バグ修正
* `enhancement` - 新機能
* `refactor` - リファクタリング
* `documentation` - ドキュメント
* `ui/ux` - UI/UX関連
* `backend` - バックエンド関連
* `frontend` - フロントエンド関連
* `supabase` - Supabase関連
* `figma` - Figma Make関連
